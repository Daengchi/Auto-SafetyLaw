# 🛡️ 안전보건 법규 검토 자동화

국가법령정보센터 Open API를 활용하여 안전보건 관련 법령의 준수평가표 생성, 제·개정 모니터링, 이메일 알림을 자동화하는 시스템입니다.

---

## 시스템 구성

```
설치.bat 실행 (최초 1회)
        │
        ▼
  C:\SafetyLaw\ (또는 선택한 경로)
  ├── 01. 법규 데이터 관리\        ← 법규준수평가표 생성·업데이트
  ├── 02. 제개정사항 모니터링\     ← 법령 변경 감지
  ├── 03. 법규 제개정 이메일 알림\ ← 변경 시 이메일 발송
  ├── laws.json                    ← 모니터링 대상 법령 목록
  ├── 01_신규생성.bat              ← 수동 실행 (평가표 처음 생성)
  ├── 01_업데이트.bat              ← 수동 실행 (기존 평가표 갱신)
  ├── 02_제개정모니터링.bat        ← 수동 실행 (제개정 변경 확인·리포트 생성)
  └── 법규모니터링_자동실행.bat    ← 작업 스케줄러 자동 실행
```

**자동화 흐름 (매 평일 09:00)**

```
GitHub 최신 코드 pull
        │
        ▼
02. 제개정사항 모니터링 실행
        │  법령 변경 감지 시
        ▼
03. 이메일 알림 발송 → 담당자 수신
```

---

## 빠른 시작

| 단계 | 작업 |
|------|------|
| **1** | `설치.bat` 실행 (관리자 권한) → 설치 폴더 선택 |
| **2** | `01. 법규 데이터 관리\.env` 및 `02. 제개정사항 모니터링\.env` 에 API 키 입력 |
| **3** | `01_신규생성.bat` 실행 → `output\` 폴더에서 결과 확인 |

## 설정 파일 안내

### `laws.json` — 모니터링 대상 법령 목록

설치 경로 최상위에 위치. 메모장으로 열어 불필요한 법령을 삭제합니다.

```json
[
  "산업안전보건법",
  "산업안전보건기준에 관한 규칙",
  "중대재해 처벌 등에 관한 법률",
  "위험물 안전관리법",
  "고압가스 안전관리법",
  "소방시설 설치 및 관리에 관한 법률",
  "화재의 예방 및 안전관리에 관한 법률",
  "화학물질관리법",
  "잔류성오염물질 관리법",
  "환경오염시설의 통합관리에 관한 법률",
  "화학물질의 등록 및 평가 등에 관한 법률",
  "폐기물관리법",
  "물환경보전법",
  "대기환경보전법",
  "토양환경보전법"
]
```

### `.env` (01, 02 폴더) — API 키

```
LAW_API_KEY=발급받은_인증키
```

> API 키(OC) 발급: https://open.law.go.kr/LSO/main.do  
> 발급·사용 시 **호출 PC의 공인(외부) IP 주소**를 법령센터에 등록해야 합니다.
> - **확인 방법**: 해당 PC 브라우저에서 네이버에 `내 아이피 주소 확인` 검색 → 표시되는 IP 주소
> - ⚠ `ipconfig`로 나오는 내부 IP(`192.168.x.x`, `10.x.x.x` 등)는 **등록하면 안 됩니다.** 외부에서 보이지 않으므로 검증이 실패합니다. 반드시 위 방법으로 확인한 **공인 IP**를 등록하세요.
> - 미등록 시 `사용자 정보 검증에 실패하였습니다` 오류로 법령 검색 결과가 0건이 되어 "법령을 찾을 수 없습니다"로 표시됩니다.
> - 사업장(PC)마다 공인 IP가 다르므로, **새 PC에서 실행하려면 그 PC의 공인 IP를 법령센터에 추가 등록**해야 합니다. 회사 네트워크가 여러 공인 IP로 나가거나 IP가 수시로 바뀌면 사내 IT팀에 고정 공인 IP를 문의하세요.

### `.env` (03 폴더) — SMTP 이메일 설정

```
SMTP_HOST=사내메일서버주소
SMTP_PORT=25
SMTP_USER=
SMTP_PASSWORD=
SMTP_FROM=발신자이메일
SMTP_USE_TLS=false
```

  "담당자1@회사.com",
  "담당자2@회사.com"
]
```

## 사용 방법

### `01_신규생성.bat` — 법규준수평가표 처음 만들기

- **언제**: laws.json 에 정의된 전체 법령의 평가표를 처음 생성할 때
- **결과**: `01. 법규 데이터 관리\output\법규준수평가표_YYYYMMDD.xlsm`

### `01_업데이트.bat` — 기존 평가표 갱신

- **언제**: 이미 생성된 평가표를 최신 법령 정보로 업데이트할 때
- **결과**: 파일 선택창 → 선택한 파일을 최신 정보로 덮어씀

### `02_제개정모니터링.bat` — 제개정 변경 확인·리포트 생성

- **언제**: laws.json 법령의 제·개정 변경 여부를 수동으로 확인하고 Excel 리포트를 받고 싶을 때
- **결과**: `02. 제개정사항 모니터링\output\법규_제개정_모니터링_YYYYMMDD_HHMM.xlsx`
- 화면에 법령별 진행 상황(신규/개정/변경 없음)이 표시됩니다. 이메일은 발송하지 않습니다.

### `법규모니터링_자동실행.bat` — 자동 모니터링

- **언제**: 작업 스케줄러가 자동 실행 (매 평일 09:00)
- **결과**: 법령 변경 감지 시 `recipients.json`의 모든 수신자에게 이메일 발송
- **로그**: `logs\실행로그.log`

> 모든 bat 파일은 실행 시 자동으로 `git pull`하여 최신 코드를 반영합니다.

## 자동화 설정 (작업 스케줄러)

설치 중 `Register Task Scheduler? [Y/N]` 질문에 **Y** 를 선택하면 자동 등록됩니다.

수동 등록이 필요한 경우:
1. Windows 작업 스케줄러 열기
2. 새 작업 만들기 → 트리거: 매주 월~금 09:00
3. 동작: `[설치경로]\법규모니터링_자동실행.bat` 실행

```

2. 사업장 PC는 다음 bat 파일 실행 시 자동으로 git pull 수행
   → 코드 재배포 불필요
