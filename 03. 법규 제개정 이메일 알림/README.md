# 법규 제개정 이메일 알림

`laws.json`에 등재된 법규를 정기 모니터링하여 **개정이 발생하면** 주소록의 전자메일 주소로
어떤 법이 / 몇 개 조항이 / 무슨 내용이 개정됐는지 자동 발송하는 서비스입니다.

개정 **감지**는 옆 프로젝트 **02. 제개정사항 모니터링**이 담당합니다. 02가 실행되면서
개정분을 공유 루트의 `amendments.json`으로 출력하고, 본 프로젝트(03)는 그 파일을 읽어
이메일을 작성·발송합니다. (감지 로직은 03에 없습니다 — 단일 진실 공급원은 02)

```
02. 모니터링  ──(amendments.json)──▶  03. 이메일 알림  ──(사내 SMTP)──▶  수신자
```

---

## 사전 요구사항

- **Python 3.10 이상**
- **사내 SMTP 서버** 접속 정보 (호스트/포트/계정)
- **02. 제개정사항 모니터링**이 먼저 실행되어 `amendments.json`이 생성되어 있어야 함

---

## 설치 및 설정

### 1. 패키지 설치

```bash
pip install -r requirements.txt
```

(`smtplib`·`email`·`json`은 파이썬 표준 라이브러리이므로 `python-dotenv`만 설치됩니다.)

### 2. SMTP 설정 — `.env`

`.env.example`를 복사해 사내 SMTP 정보를 채웁니다.

```bash
copy .env.example .env
```

| 변수 | 설명 |
|------|------|
| `SMTP_HOST` | 사내 메일 서버 호스트 (필수) |
| `SMTP_PORT` | 포트 (기본 25) |
| `SMTP_USER` | 인증 계정 (서버가 인증을 요구할 때만) |
| `SMTP_PASSWORD` | 계정 비밀번호 |
| `SMTP_FROM` | 발신자 주소 (필수) |
| `SMTP_USE_TLS` | STARTTLS 사용 시 `true` |

### 3. 수신자 — `recipients.json`

알림을 받을 이메일 주소 목록(단일 목록)입니다.

```json
[
  "hong@corp.com",
  "kim@corp.com"
]
```

## 실행

```bash
python main.py            # 개정 있으면 메일 발송
python main.py --dry-run  # 발송하지 않고 제목/본문만 콘솔 출력 (테스트)
```

- `amendments.json`의 `개정` 배열이 비어 있으면 "발송 대상 없음"으로 종료합니다.

## 정기 실행 (스케줄링)

**평일(월~금) 오전 9시**에 02(점검) → 03(발송)을 자동 실행하도록 Windows 작업 스케줄러에 등록합니다.
관련 파일은 상위 공유 폴더(`09. 안전보건 법규 검토 자동화\`)에 있습니다.

- `법규모니터링_자동실행.bat` — 02 점검 후 03 발송을 순서대로 실행. 모든 출력은 `logs\실행로그.log`에 누적.
- `작업등록.ps1` — 위 배치 파일을 작업 스케줄러에 등록하는 스크립트.

### 등록 (PowerShell에서 1회)

```powershell
cd "d:\김용희\'26년 업무\09. 안전보건 법규 검토 자동화"
.\작업등록.ps1
```

### 확인 / 테스트 / 해제

```powershell
# 등록 상태 확인 (Ready, 트리거 월~금 09:00)
Get-ScheduledTask -TaskName "법규제개정_이메일알림"

# 지금 즉시 실행해보기 (logs\실행로그.log 와 메일 확인)
Start-ScheduledTask -TaskName "법규제개정_이메일알림"

# 작업 해제(삭제)
Unregister-ScheduledTask -TaskName "법규제개정_이메일알림" -Confirm:$false
```

> **참고**
> - 작업 스케줄러는 **PC가 켜져 로그인된 상태**에서만 실행됩니다(서버가 아닌 데스크톱 기준).
> - 9시에 PC가 꺼져 있어 실행을 놓쳐도, `-StartWhenAvailable` 설정으로 **다음에 PC를 켜면 따라잡아 실행**됩니다.
>   또한 개정 감지는 스냅샷 비교 방식이라, 놓친 개정도 다음 실행에서 그대로 감지·발송됩니다(중복 없이 1회).
